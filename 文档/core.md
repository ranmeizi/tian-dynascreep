# 核心思路

代码只在1次循环中跑，memo是比较重要的
所以需要把要干的事写在 memo 中

在执行者之上，抽象出管理机构，负责整个甜朝的管理

## 顺序

- 策略阶段 各个抽象部门发动策略技能，有可能对单位施加任务
- 行动阶段 执行各个 单位 的动作
